描述
10
       rpm 是一个很有用的包管理器, 可以用于生成, 安装, 查询, 核实, 更新以及卸载单个软件包. 一个包通常包括一个文件
11
       档以及关于包的信息, 比如名字, 版本, 描述等.
12
13
       必须选取下列基本模式中一个: 初始化数据库, 重新生成数据库, 生成包, 重新编译包, 从Tar包生成RPM包, 查询, 显示
14
       查询标签, 安装, 更新, 卸载, 核实, 签名检查, 再签名, 添加签名, 设置所有者和组 以及 显示配置.

       --ftpport <port>, --httpport <port>

安装和升级选项
83
       通常的rpm安装命令如下:
84
85
            rpm -i [install-options] <package_file>+
86
87
       这样将安装一个新的包. 通常的rpm升级命令如下:
88

            rpm -U [install-options] <package_file>+
90
91
       这样将当前已安装的包升级或新安装到一个新的RPM版本. 升级和安装是一样的, 区别在于升级要将所有别的版本的包 从
92
       系统移去.
93
94
            rpm [-F|--freshen] [install-options] <package_file>+
95
96
       这样也升级包, 但只有在系统存在一个更早版本的包时候才使用这种方式.
97
98
       如果包在安装之前要先进行下载, <package_file>应该指明为一个 ftp或者http URL. 要得到关于RPM包生成的ftp和http
99
       支持的信息, 可以参阅FTP/HTTP 选项.

103

115

117

146

       --ignoreos

151

              将文件从<oldpath>改置到<newpath>, 这使得可以浮动包.

查询选项
164

168
       可以制定包信息的打印格式, 这时要用[--queryformat|-qf]选项, 格式串跟在选项后面.

172
       的名字取代, 该名字被括在{}之间. 而且tag的RPMTAG_部分也被忽略.
173
174

生成选项
332
       The general form of an rpm build command is
333
334
           rpm -[b|t]O [build-options] <package_spec>+
335
336
       The argument used is -b if a spec file is being used to build the package and -t if RPM should look inside
337
       of a gzipped (or compressed) tar file for the spec file to use. After the first argument, the  next  argu-
338
       ment (O) specifies the stages of building and packaging to be done and is one of: rpm通常的生成选项如下:
339
340
           rpm -[b|t]O [build-options] <package_spec>+
341
342
       -bp    Executes  the  "%prep"  stage from the spec file.  Normally this involves unpacking the sources and
343
              applying any patches.
344
345
       -bl    Do a "list check".  The "%files" section from the spec file is macro expanded, and checks are  made
346
              to verify that each file exists.
347
348
       -bc    Do the "%build" stage from the spec file (after doing the prep stage).  This generally involves the
349
              equivalent of a "make".
350
351
       -bi    Do the "%install" stage from the spec file (after doing the prep and build stages).  This generally
352
              involves the equivalent of a "make install".
353
354
       -bb    生成一个二进制包(在完成prep, build, 以及 install阶段之后)
355
356
       -bs    只生成源包(在完成prep, build, 以及 install阶段之后)
357
358
       -ba    生成二进制包和源包(在完成prep, build, 以及 install阶段之后)
359

       RPM包括简单的FTP和HTTP客户端程序,这样可以简化那些从internet取得的包的安装和查询.  用于安装,升级,以及查询操

       如 果忽略掉:password部分,会提示要求给出口令(每个用户/主机名对一次) 如果用户名和口令都忽略了,就使用匿名ftp.
